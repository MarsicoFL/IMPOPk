impopₖ — Tutorials

This guide walks through the four inference modes of impopk using the precomputed example datasets shipped in data/examples/. Each section ends with a ready-to-run command.

What impopₖ does in one sentence. Given a windowed pairwise identity table produced from a pangenome alignment, it infers IBD segments (2-state HMM), local ancestry tracts (N-state HMM with softmax emissions), and kinship coefficients (from the detected IBD segments).

Binaries

BinaryPurpose
ibsWrapper over impg similarity. Produces the windowed pairwise identity TSV that all downstream tools consume.
ibd2-state HMM for IBD segment detection.
ibd-validateCompare detected IBD against a gold-standard set (e.g. msprime ground truth).
ancestryN-state HMM for local ancestry inference.
jacquardJacquard δ coefficient estimation from IBD segments.

Building

git clone https://github.com/MarsicoFL/IMPOPk.git
cd IMPOPk
cargo build --release

All five binaries will be in target/release/.

Pipeline overview

Pangenome alignment (PAF) + assemblies (AGC)
              │
              │   ibs  (wraps impg similarity)
              ▼
Windowed pairwise identity TSV  ◄── input/ibs.tsv in each example
              │
     ┌────────┼──────────────┐
     ▼        ▼              ▼
    ibd   ancestry      (IBS enrichment scan,
     │      │            awk-only, see ibs/run.sh)
     ▼      ▼
  IBD segs  painted tracts
     │
     ▼
  jacquard  → kinship δ coefficients

Mode 1 — IBS enrichment

Summarise the fraction of within-population pairs with high identity in each window. No HMM is needed; the input TSV already carries the signal. 

cd data/examples/ibs
bash run.sh

Output (output/ibs_enrichment.tsv): one row per 10 kb window with n_pairs, n_ibs, and frac_ibs. Regions of elevated frac_ibs relative to the genome-wide mean indicate haplotype sharing or selection signals (the paper's chr2 LCT scan is built from exactly this quantity).

Mode 2 — IBD detection (2-state HMM)

For each pair of haplotypes, decode windows into IBD vs non-IBD using Gaussian emissions and a forward-backward + Viterbi decode. Parameters are initialised from population priors and refined via Baum-Welch. 

cd data/examples/ibd
bash run.sh

Output (output/ibd.tsv): one row per detected IBD segment, with coordinates, length in windows, mean identity, posterior confidence, and LOD score.

The default minimum segment length is 2 Mb (configurable with --min-len-bp). The EUR population prior is used here because the example panel is EUR-only; use --population Generic when your panel is mixed.

Mode 3 — Local ancestry (N-state HMM)

For each query haplotype and K reference populations, per-window emissions are computed as the maximum identity to any reference haplotype in each population, then converted to probabilities via a softmax with adaptive Bradley-Terry temperatures. 

cd data/examples/ancestry
bash run.sh

Input files:

  • input/ibs.tsv — precomputed pairwise identity TSV
  • input/populations.tsv — TSV: population haplotype_id
  • input/queries.txt — one query haplotype per line

Output (output/ancestry.tsv): painted ancestry tracts per query haplotype with posterior confidence and discriminability scores.

Mode 4 — Founder painting (pedigree)

The same ancestry binary is used. Instead of populations, each "population" slot is one grandparent haplotype, so the HMM paints each descendant haplotype with its founder of origin. 

cd data/examples/pedigree
bash run.sh

In input/populations.tsv the four rows are four individual founders (e.g. GP1, GP2, GP3, GP4). The paper's CEPH 1463 validation runs this exact configuration with the four grandparents of the platinum pedigree.

Kinship from detected IBD

jacquard estimates the nine Jacquard condensed identity coefficients (here, their deltas) for a pair of diploid individuals from the IBD detected across the four haplotype pairings: 

jacquard \
  --ibs   ibs_chr12.tsv \
  --hap-a1 HG00097#1  --hap-a2 HG00097#2 \
  --hap-b1 HG00099#1  --hap-b2 HG00099#2

Running on your own pangenome

If you have a PAF alignment and an AGC file, the full pipeline is: 

# 1. Windowed pairwise identity
ibs -a                      data/alignments/hprc_chr12.paf.gz \
    --sequence-files        data/assemblies/HPRC_r2.agc \
    -r                      CHM13 \
    --region                chr12:1-133324548 \
    --size                  10000 \
    --subset-sequence-list  panel_subset.txt \
    --threads               8 \
    --output                ibs_chr12.tsv

# 2a. IBD
ibd --similarity-file ibs_chr12.tsv \
    --region chr12:1-133324548 --region-length 133324548 \
    --size 10000 --min-len-bp 2000000 \
    --threads 8 --output ibd_chr12.tsv

# 2b. Local ancestry
ancestry --similarity-file ibs_chr12.tsv \
         --window-size 10000 \
         --populations populations.tsv \
         --query-samples queries.txt \
         --emission-model max --estimate-params \
         --threads 8 --output ancestry_chr12.tsv

See data/examples/README.md for detail on input/output formats.
