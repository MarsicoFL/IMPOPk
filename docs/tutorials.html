Tutorial 3: IBS Computation Core

What is IBS?

Identity-by-State (IBS) measures sequence similarity between two haplotypes over a genomic window. Unlike variant-based methods that define identity per SNP or indel, impopk defines IBS per genomic segment: the identity of a window is the fraction of aligned bases where both haplotypes agree, computed over the full window size.

Pairwise identity can be extracted from three sources:

  • ibs-from-paf — directly from PAF alignment CIGAR strings (recommended for simplicity)
  • ibs — via impg implicit pangenome queries (legacy, requires impg + AGC)
  • ibs-from-tpa — from pre-indexed TPA format with spatial index (fastest for regional queries)

Quick Start

# Compute IBS for 1 Mb of chr12 (10 kb windows)
ibs-from-paf \
    -a data/alignments/hprc465vschm13.aln.paf.gz \
    -r CHM13 \
    --region chr12:1000000-2000000 \
    --size 10000 \
    --output ibs_chr12_1Mb.tsv

Output Format

chrom  start    end      group.a    group.b    estimated.identity  group.a.length  group.b.length
chr12  1000000  1009999  HG00097#1  HG00099#1  0.998400            10000           9987
chr12  1000000  1009999  HG00097#1  HG00100#2  0.999100            10000           10000
chr12  1010000  1019999  HG00097#1  HG00099#1  0.998900            10000           10000

Identity Value Ranges

Typical pairwise identity values at 10 kb windows for HPRC pangenome data:

Population pairMean identityStd dev
Within-population (e.g., EUR-EUR)~0.9992~0.0012
Cross-population (e.g., EUR-AFR)~0.9988~0.0015
Same individual (hap1 vs hap2)~0.9993~0.0010
Note

Identity values are very high (>0.996) because pangenome haplotypes share recent common ancestry. Small differences (<0.001) in mean identity between populations are the signal that downstream HMMs use for ancestry and IBD inference.

Fast Mode: IBS from TPA (recommended for regional queries)

For repeated or regional queries, pre-indexing with the TPA format avoids re-scanning the full PAF file. This gives 6x speedup for typical single-chromosome queries.

One-time setup

# Step 1: Convert PAF to TPA (requires cigzip: github.com/AndreaGuarracino/cigzip)
cigzip encode --paf alignments.paf.gz | cigzip compress -i - -o alignments.tpa

# Step 2: Build spatial index (~15 seconds for full genome)
tpa-spatial-index --tpa alignments.tpa --output alignments.sidx

Regional queries (seconds instead of minutes)

ibs-from-tpa \
    --tpa alignments.tpa \
    --spatial-index alignments.sidx \
    --agc assemblies.agc \
    --region chr12:1000000-2000000 \
    --size 10000 \
    --output ibs_chr12_1Mb.tsv
Drop-in replacement

Output of ibs-from-tpa is numerically equivalent to ibs-from-paf (validated: mean diff = 0.00006, max diff = 0.008, due to CIGAR reconstruction from tracepoints). It can be used directly with ancestry and ibd without any changes.

When to use which

ibs-from-paf: Simple setup, no pre-processing needed. Best for one-off full-genome runs.
ibs-from-tpa: Requires TPA conversion (one-time, ~2 min) + AGC file. Best for interactive analysis, repeated regional queries, or when querying many small regions.

Filtering by Population

Tutorial 4: IBD Detection Core

The 2-State HMM

Hidden Markov Model for IBD Detection

Identity Distributions: IBD vs Non-IBD

The HMM discriminates two overlapping distributions

The identity gap (~0.0007) between IBD and non-IBD pairs is the signal the HMM uses. Baum-Welch estimates these emission distributions from the data.

Tutorial 5: Ancestry Inference Core

N-State HMM for Local Ancestry

Each genomic window is assigned to the ancestral population it most likely derives from. States = populations (e.g., AFR, EUR, AMR). Emissions are computed from pangenome identity between query and reference haplotypes.

Input Preparation

Populations file (TSV)

Two columns, no header: POPULATION_NAME<tab>HAPLOTYPE_ID. One line per haplotype (not per individual).

# Example: populations.tsv
# EUR	HG00097#1
# EUR	HG00097#2
# AFR	HG01884#1
# AFR	HG01884#2

# Generate from sample lists
for pop in AFR EUR AMR; do
    while read -r hap; do
        printf '%s\t%s\n' "$pop" "$hap"
    done < "data/samples/${pop}.txt"
done > populations.tsv

Query samples file

One haplotype ID per line (e.g., HG00639#1). These are the samples to classify. They should NOT overlap with reference haplotypes in the populations file.

Compute similarities

# Create subset with query + reference haplotypes
cat data/samples/AFR.txt data/samples/EUR.txt data/samples/AMR.txt query_amr.txt \
    | sort -u > ancestry_subset.txt

ibs-from-paf \
    -a data/alignments/hprc465vschm13.aln.paf.gz \
    -r CHM13 --region chr12:1-133000000 --size 5000 \
    --subset-sequence-list ancestry_subset.txt \
    --output similarities_chr12.tsv

Run with Auto-Configure (Recommended)

ancestry \
    --similarity-file similarities_chr12.tsv \
    --populations populations.tsv \
    --query-samples query_amr.txt \
    --auto-configure \
    --identity-floor 0.9 \
    -o ancestry_chr12.tsv \
    --posteriors-output posteriors_chr12.tsv
Auto-Configure

Automatically estimates temperature, pairwise weight, emission context, and enables pairwise contrast emissions. Eliminates manual tuning.

Pairwise Contrast: The Key Innovation

Standard softmax uses a single temperature for all populations. When EUR-AMR gap is ~0.0002 but EUR-AFR gap is ~0.003, a single temperature can't handle both. Pairwise contrast emissions calibrate per-pair temperatures via Bradley-Terry scoring.

Workflow

Step 0: Align Platinum assemblies to CHM13

The Platinum Pedigree samples are NOT in HPRC. You must align their assemblies to CHM13 first:

# Download Platinum assemblies from S3
for sample in NA12891 NA12892 NA12889 NA12890 NA12879 NA12881 NA12882 NA12885 NA12886; do
    aws s3 cp --no-sign-request \
        s3://platinum-pedigree-data/assemblies/${sample}/ ./${sample}/ --recursive
done

# Align each assembly to CHM13 reference
for fa in */assembly/*.fa.gz; do
    minimap2 -cx asm5 --eqx -t 8 chm13.fa "$fa"
done > platinum.paf

Step 1: Compute similarities

# platinum_subset.txt: one haplotype per line (e.g., NA12891#1)
# 18 haplotypes: 4 grandparents x 2 haps + 5 offspring x 2 haps
ibs-from-paf \
    -a platinum.paf \
    -r CHM13 --region chr1:1-248387328 --size 5000 \
    --subset-sequence-list platinum_subset.txt \
    --output platinum_similarities.tsv

Step 2: Run 4-state founder painting

# grandparents.tsv: maps each grandparent haplotype to a "founder" label
# Format: FOUNDER_NAME<tab>HAPLOTYPE_ID (e.g., NA12891<tab>NA12891#1)
ancestry \
    --similarity-file platinum_similarities.tsv \
    --populations grandparents.tsv \
    --query-samples offspring.txt \
    --auto-configure --identity-floor 0.9 \
    -o pedigree_painting.tsv \
    --posteriors-output pedigree_posteriors.tsv

Key Insight

Chimeric assemblies are stitched from real HPRC sequences at msprime-defined boundaries. This preserves realistic alignment artifacts and population-specific identity patterns that purely simulated sequences would miss.

Simulation Code

Step 1: Simulate ancestry tracts with msprime

import msprime
import numpy as np

# 2-way admixture: 70% AFR, 30% EUR on chr12 (133 Mb)
demography = msprime.Demography()
demography.add_population(name="AFR", initial_size=14000)
demography.add_population(name="EUR", initial_size=9000)
demography.add_population(name="ANC", initial_size=18500)
demography.add_population(name="ADM", initial_size=30000)
demography.add_admixture(time=15, derived="ADM", ancestral=["AFR","EUR"], proportions=[0.7,0.3])
demography.add_population_split(time=2000, derived=["AFR","EUR"], ancestral="ANC")

ts = msprime.sim_ancestry(
    samples={"ADM": 5},  # 5 admixed diploids → 10 haplotypes
    demography=demography,
    sequence_length=133_000_000,
    recombination_rate=1e-8,
    random_seed=42
)

# Extract ground-truth ancestry tracts using link_ancestors
# Each sample's lineage is traced through the tree to the source population
tables = ts.dump_tables()
ancestors = tables.link_ancestors(
    samples=ts.samples(),
    ancestors=np.where(np.array([ts.node(n).population in [0, 1]  # AFR=0, EUR=1
                                  for n in range(ts.num_nodes)]))[0]
)
# For each (sample, ancestor) pair, the ancestor's population gives local ancestry
# Group by sample and merge adjacent intervals with same ancestry to get tracts
# Output: BED file with (chrom, start, end, population) per haplotype

Step 2: Stitch chimeric assemblies from real HPRC sequences

# For each simulated haplotype, replace tracts with real HPRC FASTAs
# AFR tracts → real AFR assembly sequence (e.g., HG01884)
# EUR tracts → real EUR assembly sequence (e.g., HG00097)

# Extract sequences from AGC
agc getctg assemblies.agc HG01884#1#CM094060.1 > afr_donor.fa
agc getctg assemblies.agc HG00097#1#CM094060.1 > eur_donor.fa

# Stitch at msprime tract boundaries using samtools faidx + cat

Step 3: Align and run impopk

# Align chimeric assemblies to CHM13
minimap2 -cx asm5 --eqx -t 8 chm13.fa chimeric_haps.fa > chimeric.paf

# Combine chimeric + reference panel PAFs
cat chimeric.paf reference_panel.paf > combined.paf

# Compute IBS and run ancestry inference

    --query-samples sim_query.txt --auto-configure -o sim_ancestry.tsv

Tutorial 8: Advanced Features Expert

1. eGRM: Assembly-Direct Genetic Relationship Matrix

# Haploid eGRM (GCTA-compatible)
ancestry \
    --similarity-file similarities.tsv \
    --populations populations.tsv --query-samples query.txt \
    --output-egrm egrm_chr12 --center-egrm \
    -o ancestry.tsv

# Diploid eGRM (for GWAS/heritability)
ancestry \
    --similarity-file similarities.tsv \
    --populations populations.tsv --query-samples query.txt \
    --output-egrm egrm_diploid --diploid-egrm --double-center-egrm \
    -o ancestry.tsv

# Use with GCTA
gcta64 --grm egrm_diploid --pca 10 --out pca_results

2. Demographic Inference

ancestry \
    --similarity-file similarities.tsv \
    --populations populations.tsv --query-samples query_amr.txt \
    --auto-configure --identity-floor 0.9 \
    --demographic-inference --demographic-output demographics.tsv \
    --max-pulses 3 \
    -o ancestry.tsv
How It Works

Fits a shifted exponential mixture model to ancestry tract lengths. Each pulse represents an admixture event. Rate encodes generations since admixture. BIC selects the optimal number of pulses.

3. Cross-Chromosome Parameter Transfer

# Learn params on a clean chromosome
ancestry --similarity-file sim_chr12.tsv \
    --populations pop.tsv --query-samples q.txt \
    --auto-configure --identity-floor 0.9 \
    --save-params params_chr12.json -o /dev/null

# Apply to a noisier chromosome
ancestry --similarity-file sim_chr1.tsv \
    --populations pop.tsv --query-samples q.txt \
    --load-params params_chr12.json -o ancestry_chr1.tsv
